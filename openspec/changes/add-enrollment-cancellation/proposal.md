# 提案：新增报名取消能力

## 目标

允许用户主动取消自己处于排队中或未支付状态的报名。这项能力可以验证系统在并发场景下的库存补偿逻辑，避免在 Sprint 3 的高压测试中出现库存被错误放大或长时间锁死的问题。

## 范围

包含范围：
- 为 `QUEUING` 和 `PENDING` 状态提供主动取消接口 `POST /enrollments/:id/cancel`。
- 确保主动取消时能够安全释放 Redis 临时锁，并正确回补 MySQL 活动库存。
- 提供基础前端入口，例如“取消排队”或“取消未支付订单”。
- 验证主动取消与过期扫描任务 `ScanExpired` 并发时的竞态安全。

不包含范围：
- 不处理 `PAID` 订单的退款，延期到 Sprint 4。
- 不处理更高层级的角色管理或商户通知逻辑。
