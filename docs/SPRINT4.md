## 一、核心交付任务清单 (Sprint 4)

完成以下三项核心收口任务：

### 1. 万级数据构建与后端性能调优
* **基础数据注入**：编写轻量级造数脚本（如 `cmd/seeder`），向 MySQL 数据库中注入不低于 **1万条真实数据**（包括用户、活动、订单及相关关联记录），模拟真实业务环境。
* **索引与响应调优**：在万级数据背景下进行接口调用，针对活动列表、订单列表等分页查询接口，补齐必要的 MySQL 索引，消除全表扫描，确保所有关键 API 的响应耗时稳定在 200ms 以内。

### 2. 前端真实对接与流畅度优化
* **关闭 Mock 联调**：彻底关闭 `VITE_USE_MOCK` 配置，确保所有前端页面均对接真实的后端服务和万级数据库。
* **分页与流畅度处理**：针对活动广场、我的订单等列表页面，确认分页（Pagination）或懒加载（Lazy Load）机制生效，防止大量数据一次性下发导致前端浏览器卡顿。
* **交互防连击保护**：核心业务按钮（如报名排队、确认支付、自主取消）加上 Loading 状态锁，进行物理防抖，避免用户连击造成重复请求。

### 3. 全链路端到端功能走查
* **用户视角完整链路走查**：从真实商户及用户视角，手动走通核心闭环流程，确保无控制台报错：
  `商户登录 -> 创建并发布活动 -> C端用户浏览 -> 报名排队 -> 创单/支付/主动取消 -> 接收对应系统通知`。
* **数据精确性校对**：在常规业务流程操作后，校验 MySQL 和 Redis 缓存中的库存扣减与回补完全一致，确保状态转移逻辑无瑕疵。

## 二、最终验收标准 (DoD)

- **数据承载**：数据库包含不低于 10,000 条有效测试数据，所有分页接口响应速度保持在 200ms 以内。
- **全链路闭环**：用户从注册登录到报名、买单、取消、接收通知的端到端交互 100% 跑通，前端界面无卡顿、无白屏，控制台无报错。
- **数据一致性**：在用户视角的常规业务操作下，数据库与缓存中的库存数据完全一致，无状态丢失或超卖。
